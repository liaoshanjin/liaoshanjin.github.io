windows 设置 Go 的开发环境

在 Windows 中开发的时候，本人不喜欢将代码放在 C 盘， 同时也不喜欢将软件安装在 C 盘。会单独分出一个盘专门放代码 和 软件。以此为背景。

安装 GoLang

其实 win 上安装一个软件很傻瓜。一路下去就好了。只是我这边没有安装在 C 盘。而是安装在 E 盘 的 GoLang 目录下。

理解 GOPATH 和 GOROOT
  • GOROOT 目录是我们go语言所在目录，已 win 为例：就是指 go.exe 文件所在的目录。
  • GOPATH 在go 1.11版本之前，我们通常都是设置 gopath 目录来存放一些我们自定义的包。
配置环境变量
go env -w GOROOT=E:\GoLang
go env -w GOPATH=D:\GoLang\GOPATH
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy/,direct
go env -w GO111MODULE=on

主要设置 GOROOT 、GOPATH、GOPROXY 、GO111MODULE 这四个环境变量

安装 GoLand

安装这个软件也很傻瓜。我们也将他安装在 软件盘 下 GoLand 目录，也就是 ：E:\GoLand 。

开始开发

我们开启了 gomodule:

在项目中执行 ：

go mod init projectName

然后在引用自定义包的时候，包名需要和路径名称一致，其他正常开发。

单元测试

例如:

simplemath/add.go

package simplemath

func Add(a, b int) int {
	return a + b
}

很简单的一个方法，写一个单元测试：

创建单元测试文件： add_test.go

package simplemath

import "testing"

func TestAdd(t *testing.T) {
	r := Add(1, 2)
	if r != 3 {
		t.Errorf("Add(1, 2) failed. Got %d, expected 3.", r)
	}
}

执行测试：

cd xxx/simplemath

go test

其中：

  • 测试文件名必须以 “_test.go” 结尾；
  • 测试函数名必须以 “TestXxx” 开始；
  • 命令行下使用 “go test” 即可启动测试；
