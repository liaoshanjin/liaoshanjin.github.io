使用 Laravel 的 API资源开发接口

API 资源介绍

在使用 Laravel 开发接口的时候，很多时候我们需要使用到 transform 层，转换一下，我们想要显示给客户端的数据。

在 Laravel5.5 之前的版本中，我们可以使用 league/fractal 包解决。典型的就是 dingo/api 了。

但是在 Laravel5.5 以及之后的版本中， Laravel 框架自带了 Api 资源 ，接下来我们就来使用这个资源开发。

新建资源

新建资源很简单。我们只需要使用 Laravel 框架的脚手架就能完成。

php artisan make:resource RoleResource

就能在目录下看到文件 app\Http\Resources\RoleResource.php。

<?php

namespace App\Http\Resources;

use Illuminate\Http\Resources\Json\JsonResource;

class RoleResource extends JsonResource
{
    /**
     * Transform the resource into an array.
     *
     * @param  \Illuminate\Http\Request  $request
     * @return array
     */
    public function toArray($request)
    {
        return parent::toArray($request);
    }
}
使用资源

新建好了资源，接下来我们该怎么使用呢？和 league/fractal 差不多，当我们查询的数据是一个 item 时，我们使用 new 一个资源。

..
..

public function show(Role $role)
{
    return new RoleResource($role);
}

而查询结果是一个集合或者 paginator 的时候该怎么办呢？

当查询结果是集合的时候:

public function index(Request $request)
{
    $lists = Role::query()->get();

    return RoleResource::collection($lists);
}

结果如下： 假如我现在不想让客户端的人知道我 name 的真正字段，又该如何呢？自需要在 RoleResource.php 中重新定义 toArray 方法：

.
.

public function toArray($request)
{
    return [
        'id'           => $this->id,
        'my_name'      => $this->name,
        'display_name' => $this->display_name,
        'description'  => $this->description,
    ];
}

如上所示，我们将数据库字段 name， 改成了 my_name 。同时隐藏了 created_at 和 updated_at 字段。再次请求： 如此一来我们就不会暴露数据的真正字段。同时也能添加隐藏一些字段。

扩展资源

上面的虽然能够很好的解决了部分问题。但是我们的请求毫无疑问会带上很多关联关系。以我这边的 角色 为例。每一个角色都能有很多和权限。我们能不能再请求 角色列表的时候同时显示角色所关联的权限呢？

以上收的其实 dingo/api 就有这个功能。在路由中我们 可以 添加 include 的参数。然后在 Transformers 层 定义 $availableIncludes 属性 以及 includeXXX 方法。

现在我们也来做一个这种的功能。

首先我们需要新建一个 Resource 的基类。

php artisan make:resource Resource

打开并重构里面的方法：

<?php

namespace App\Http\Resources;

use Illuminate\Http\Resources\Json\JsonResource;

/**
 * Class Resource
 *
 * @package App\Http\Resources
 */
class Resource extends JsonResource
{


    /**
     * Resource constructor.
     *
     * 添加懒加载
     */
    public function __construct($resource)
    {
        parent::__construct($resource);

        $resource->loadMissing(self::getRequestIncludes());
    }


    /**
     * 重构集合，添加懒加载
     *
     * @param mixed $resource
     *
     * @return \Illuminate\Http\Resources\Json\AnonymousResourceCollection
     * @author   : stringer <10******56@qq.com>
     * @datetime : 2019/7/3 11:10
     */
    public static function collection($resource)
    {
        $resource->loadMissing(self::getRequestIncludes());

        return parent::collection($resource);
    }


    /**
     * 获取需要懒加载的数组
     *
     * @return array
     * @author   : stringer <10******56@qq.com>
     * @datetime : 2019/7/3 11:09
     */
    public static function getRequestIncludes()
    {
        $request = request();

        if ($request->has('include')) {
            return \array_map('trim', \explode(',', trim($request->get('include'), ',')));
        }

        return [];
    }


}

这边我们重构了 __construct() 构造方法和 collection 的静态方法。正式对应的我们 Resource 的用法。

我们新增了一个 getRequestIncludes 的方法。主要用于获取路由中的 include 的参数。将这个参数用逗号分隔成数组。在使用 trim 去空。如此我们便可以得到一个数组。然后我们使用 loadMissing 方法加载数组。

如此一来我们便基本完成。例子： 可以看到我们的角色列表里面，每条角色都关联了自己的权限。

这边我们使用的是 loadMissing，他和 load 有什么区别呢？ loadMissing 其实和 load 没什么区别。只是加载过的关联不会再加载而已。

其实用法不仅仅如此。我们还可以在 RoleResource 重构 collection 或者 __construct 方法，在里面 loadMissing 更多关联。更多的用法大家可以一起探索。
