PHPunit 单元测试

最近公司写的项目也是因为测试 BUG 太多，导致进度缓慢，所以后端也开始写单元测试减少测试的压力。

我们使用的是 Laravel 框架，所以我这边就记录一下 Laravel 框架的单元测试。

单元测试 PHPunit

PHPUnit 是一个轻量级的 PHP 测试框架，Laravel 默认就支持用 PHPUnit 来做测试，并为你的应用程序配置好了 phpunit.xml 文件，只需在命令行上运行 phpunit 就可以进行测试。

创建测试文件

首先我们来创建一个单元测试文件，使用 Laravel 提供的 artisan 命令

php artisan make:test RoleApiTest

该命令会在 tests/Feature 目录中创建 RoleApiTest.php 文件，我们会发现 tests 目录中有 Feature 和 Unit 两个目录，如何区分这两个目录呢？

  • Unit —— 单元测试是从程序员的角度编写的。它们用于确保类的特定方法执行一组特定任务。
  • Feature —— 功能测试是从用户的角度编写的。它们确保系统按照用户期望的那样运行，包括几个对象的相互作用，甚至是一个完整的 HTTP 请求。

我们测试 API 属于功能测试，应该创建在 Feature 目录中。

测试添加会员

打开我们刚刚生成 RoleApiTest.php 文件

<?php

namespace Tests\Feature;

use Tests\TestCase;
use Illuminate\Foundation\Testing\WithFaker;
use Illuminate\Foundation\Testing\RefreshDatabase;

/**
 * Class RoleApiTest
 *
 * @package Tests\Feature
 */
class RoleApiTest extends TestCase
{

    /**
     * 添加角色测试
     *
     * @author   : stringer <10******56@qq.com>
     * @datetime : 2019/7/4 10:34
     */
    public function testStoreRole()
    {
        $data = [
            'name'         => 'root',
            'display_name' => 'root管理员',
            'description'  => '拥有一切权限',
        ];

        $response = $this->json('POST', '/api/v1/admin/roles', $data);

        $assertData = [
            'id'           => 2,
            'name'         => 'root',
            'display_name' => 'root管理员',
            'description'  => '拥有一切权限',
        ];

        $response->assertStatus(201)->assertJsonFragment($assertData);
    }


}

RoleApiTest 就是测试添加一个角色。使用 $this->json 可以方便的模拟各种 HTTP 请求：

  • 第一个参数 —— 请求的方法，添加角色使用的是 POST 方法；
  • 第二个参数 —— 请求地址，请求 /api/v1/admin/roles；
  • 第三个参数 —— 请求参数，传入 name，display_name，description，这三个必填参数；
  • 第四个参数 —— 请求 Header，可以直接设置 header，也可以利用 withHeaders方法达到同样的目的；

查看数据库

在上面过程中我们进行了一个最基本的测试。因为我这个测试是没有使用登录账号测试的，在工作中，很多操作都是需要登录用户才能够测试的，那么接下来我们就来改动一下上面的代码。

<?php

namespace Tests\Feature;

use http\Client\Curl\User;
use Tests\TestCase;
use Illuminate\Foundation\Testing\WithFaker;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\Traits\ActingJWTUser;

/**
 * Class RoleApiTest
 *
 * @package Tests\Feature
 */
class RoleApiTest extends TestCase
{

    use ActingJWTUser;


    /**
     * @var
     */
    protected $user;


    /**
     * 设置用户
     *
     * @author   : stringer <10******56@qq.com>
     * @datetime : 2019/7/4 10:45
     */
    public function setUp()
    {
        parent::setUp();
        $this->user = \App\Models\User::query()->first();
    }


    /**
     * 添加角色测试
     *
     * @author   : stringer <10******56@qq.com>
     * @datetime : 2019/7/4 10:34
     */
    public function testStoreRole()
    {
        $data = [
            'name'         => 'root',
            'display_name' => 'root管理员',
            'description'  => '拥有一切权限',
        ];

        $response = $this->JWTActingAs($this->user)->json('POST', '/api/v1/admin/roles', $data);

        $assertData = [
            'id'           => 6,
            'name'         => 'root',
            'display_name' => 'root管理员',
            'description'  => '拥有一切权限',
        ];

        $response->assertStatus(201)->assertJsonFragment($assertData);
    }


}

首先我们介绍一下 setUp 方法。setUp 方法会在测试开始之前执行。类似我们的构造函数。这样我们就先获取一个会员，然后以这个会员的身份进行测试。如果你需要权限，可以在设置指定的会员。

这其中我们使用了一个名为 ActingJWTUser 的 trait , 这个 trait 的主要作用是帮我们获取我们刚刚 查询出来的测试会员的 Token , 并且在我们对接口进行测试的时候可以设置 Bearer Token 。

这样我们就能方便的使用 JWTActingAs 方法，登录查询的指定用户。最后得到的响应 $response，通过 assertStatus 断言响应结果为 201，通过 assertJsonFragment 断言响应结果包含 assertData 数据。

打开我们的 RoleApiTest.php

    /**
     * 测试删除角色
     *
     * @author   : stringer <10******56@qq.com>
     * @datetime : 2019/7/4 14:19
     */
    public function testDeleteRole()
    {
        $role = $this->makeRole();

        $response = $this->actingAs($this->user)->json('DELETE', '/api/v1/admin/roles/' . $role->id);

        $response->assertStatus(204);

        $response = $this->actingAs($this->user)->json('GET', '/api/v1/admin/roles/' . $role->id);

        $response->assertStatus(404);
    }

首先通过 makeRole 创建一个角色，然后通过 DELETE 方法调用 删除角色 接口，将角色删除，断言响应状态码为 204。

接着请求角色详情接口，断言响应状态码为 404，因为该角色已经被删除了，所以会得到 404。
