在 Win 上配置 swoole 的开发环境

24 Oct 2019
@特污兔

软件说明和准备

首先 window 电脑肯定是不支持 swoole 的，所以只能用虚拟机。虚拟机我装的是 VMware 。

然后是 IDE 是 PHPstorm 。主要也是在 PHPstorm 上配置环境。

首先我在我的虚拟机上安装 LNMP 的集成环境。并且配置了一个 swoole.test 的站点。然后以下操作都是在连接上虚拟机的服务器的操作。

PHPstorm 配置 SFTP

  • 用 phpstorm 打开一个空的项目

  • 点击工具栏的 Tools => Deployment => Browser Remote Host :

  • 点击 ... 进行配置 SFTP 。

  • 输入虚拟机服务器的 IP ， SSH 端口， 账号密码等。

  • 点击 mapping ，配置 Deployment Path ,这个目录将对应我们 windows 上现在的这个项目目录。相当于 项目中的文件，通过上传可直接上传到这个配置的目录。

  • 勾选自动上传。每次修改完代码会自动上传到我们的虚拟机服务器。

  • 此时我们已经可以和服务器的文件目录同步文件了。在本地修改过的文件可以自动上传到服务器中。服务器上修改的我们也可以手动下载下来。然后我们开始配置 ssh 的终端。这样我们就能在 phpstorm 的终端使用操作服务器。点击 Tools => Start SSH session , 选择我们刚刚已经配置好的 SFTP 配置名。就可以开启一个 SSH 终端了：

  • 接下来就可以在 phpstorm 中使用 ssh 终端操作服务器了。

PHPstorm 配置 PHP 环境

如果想在 phpstorm 中运行服务器的 php 环境，该怎么做呢？

  • 首先确保虚拟机中有 PHP 环境。我这边已经装好了 LNMP 。没有问题。

  • phpstorm 中使用快捷键 ctrl + shift + s 打开设置

  • 点击 Languages & Frameworks => PHP，选择 php language level , 选择 CLI Interpreter

  • 点击加号 + => From docker,Vagrant,VM。 我们使用的是 VM 的方式。

  • 在之前我们已经配置好了一个 SFTP ，可以直接勾选。如果想替换，也可以使用其他的。

  • 一路 OK 保存。

  • 新建一个 index.php ,写入一些内容用于测试。如果上面没有配置 auto upload 可以右键选择 Deployment => Upload to x 进行上传，如果有配置则略过这步。

  • 快捷键 ctrl + shift + F10 ,即可运行成功

补充

上面的运行 php 代码只建议用于调试 php,不建议用来开启 swoole tcp 服务器等,

开启 tcp 服务器请使用上面的 ssh 终端,进行 php server.php 原生命令进行开启
